# Getting help with Mnema

The support channel for Mnema is **GitHub Issues** on this repository. There's no
separate chat, forum, or portal — the issue thread is the place.

## Before you open an issue

- **Self-hosting trouble?** Check the **Troubleshooting (self-host)** section in
  the [README](./README.md) first — the common ones (disconnected documents on
  HTTPS, ports already allocated, `PUBLIC_*` vars not taking effect, first-boot
  gating) are covered there with fixes.
- **Setup and local dev?** See the [README](./README.md) quickstart and
  [CONTRIBUTING.md](./CONTRIBUTING.md).
- **Search existing issues.** Someone may have already hit — and solved — the
  same thing.

## Opening an issue

Pick the right form when you open a new issue:

- **Bug report** — something's broken or behaving wrong.
- **Self-host / setup help** — you're stuck getting Mnema running.
- **Feature request** — you'd like Mnema to do something it doesn't.

The more detail you give (version or commit, how you're deployed, logs, exact
steps), the faster we can help.

## Response expectations

We're a small team and this is a fair-code project, so support is **best-effort**:
we aim to respond **within about a week**, usually sooner. There's no SLA beyond
that. A polite nudge on the thread is welcome if a week goes by with no word.

## Security issues

**Do not** use GitHub Issues for security vulnerabilities. Report them privately
— see [SECURITY.md](./SECURITY.md).

## Enterprise

The knowledge graph, meeting intelligence, org/IAM+SSO, audit, and
multi-workspace modules are commercial add-ons and aren't part of this repo. For
those, reach out via the website.
